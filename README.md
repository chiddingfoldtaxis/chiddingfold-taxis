# Chiddingfold Taxis — website

A fast, modern, one-page static website for **Chiddingfold Taxis** (a rebrand of
"Surrey Hill Taxis"), based in Godalming, Surrey. Built as plain HTML/CSS/JS with
**no build step** and **no runtime dependencies** — it's designed to be hosted free on
**GitHub Pages** and to load almost instantly on a phone.

> **The site owner still needs to check a few things before it goes live.**
> See **[✅ Pre-launch checklist](#-pre-launch-checklist)** — it lists every claim on the
> page that must be confirmed true (licensing, DBS, etc.) and the placeholders to fill in.

---

## Quick start

```bash
npm install          # installs dev tooling (a local server) + the OpenAI SDK for images
npm run dev          # serves the site at http://localhost:5173
```

Then open <http://localhost:5173>. That's the whole site — edit `index.html`,
`assets/css/styles.css` and `assets/js/main.js` and refresh.

## 🎨 Imagery (AI-generated or real photos)

The design looks complete out of the box with tasteful placeholders. To fill the two photo
slots (the **hero** and the **car / "why book with us"** image), you have two options:

**A) Real photos (best for trust).** Drop files into `assets/images/`:
`hero.webp` (a car on a Surrey lane) and `about.webp` (a clean, well-presented car — the
vehicle, not one named person). They'll appear automatically. A real photo of an actual
clean car reads more trustworthy than a render.

**B) Generate realistic imagery with the OpenAI API.** You run this with **your own** API key
(it's never committed):

```bash
cp .env.example .env        # then paste your OpenAI key into .env
npm run images              # generates the full image set into assets/images/
npm run images -- --list    # see all image slots
npm run images -- --only hero,airport
```

Costs a few dollars for the whole set. **Before publishing any generated image, check:**
- the car is **right-hand-drive** (wheel on the right — the model often defaults to US left-hand-drive);
- **no legible number plate** and **no fake badges/logos**;
- keep people **generic and illustrative** — fine to show a friendly driver at a distance,
  but never present an AI face as a specific named person, and don't imply a bigger fleet
  than the business has.

Additional slots the script can produce (wire them into `index.html` where you like):
`vehicle-exterior`, `vehicle-interior`, `airport`, `local-village`, `local-godalming`, `night-24-7`.
For the local village/town shots, use **real photos** — image models distort real landmarks.

Also generate/replace these referenced assets when you can: `assets/images/og-image.jpg`
(1200×630 social-share image), `apple-touch-icon.png` (180×180), and
`assets/icons/icon-192.png` / `icon-512.png` / `icon-512-maskable.png` (from the logo).

## 🚀 Deploy to GitHub Pages (custom domain `chiddingfoldtaxis.co.uk`)

1. **Push to GitHub.** Create a repo and push this folder to `main`.
2. **Enable Pages.** Repo → *Settings → Pages* → *Deploy from a branch* → `main` / `root`.
   Confirm it serves at `https://<user>.github.io/<repo>/` first.
3. **Add the custom domain.** *Settings → Pages → Custom domain* → `chiddingfoldtaxis.co.uk`
   → Save. (The `CNAME` file in this repo already contains that domain.)
4. **DNS at your registrar.** For the apex domain add the four GitHub A records and four AAAA
   records; add a `www` CNAME to `<user>.github.io`:

   | Type | Host | Value |
   |---|---|---|
   | A | @ | `185.199.108.153` |
   | A | @ | `185.199.109.153` |
   | A | @ | `185.199.110.153` |
   | A | @ | `185.199.111.153` |
   | AAAA | @ | `2606:50c0:8000::153` |
   | AAAA | @ | `2606:50c0:8001::153` |
   | AAAA | @ | `2606:50c0:8002::153` |
   | AAAA | @ | `2606:50c0:8003::153` |
   | CNAME | www | `<user>.github.io.` |

   Remove any old/parked A or URL-forwarding records. Don't use wildcard DNS.
5. **Wait for DNS**, then back in *Settings → Pages* wait for "DNS check successful", then tick
   **Enforce HTTPS** (GitHub issues a free certificate — can take up to ~24h).
6. **Done.** `www` redirects to the apex, and the apex is the canonical HTTPS host — which
   already matches the `canonical`, `sitemap.xml`, `og:url` and JSON-LD in the site.

## 📈 After launch — the biggest SEO win

The single highest-impact thing for a local taxi firm is a fully-completed **Google Business
Profile** at the Godalming address, in the right categories (Taxi service / Airport shuttle
service), with real photos and a steady trickle of **genuine Google reviews**. The website and
the profile reinforce each other. Then:

- Submit the site in **Google Search Console** and **Bing Webmaster Tools** and submit `sitemap.xml`.
- Add the live Google reviews link to the "Find us on Google" link in the reviews section of `index.html`.
- Add your real social profile URLs to the `sameAs: []` array in the JSON-LD (and the footer).

## ✅ Pre-launch checklist

The site is written to be **truthful**. A few claims and placeholders are flagged in the code
with `⚑ CONFIRM` / `TODO` comments — **check each before publishing**:

**Must be confirmed true (or removed):**
- [ ] **Licensing** — the page says *"Licensed private hire — Waverley Borough Council."*
      Confirm the current licence is held; add the operator/licence number if you have it.
      (Waverley is the correct authority for Godalming/Chiddingfold.) Appears in: trust strip,
      "Why book with us", and the footer.
- [ ] **Airport service details** — "meet you at arrivals", "return pickups", "help with luggage",
      "early-morning/overnight pickups". Keep only what's genuinely offered (airports section).
- [ ] **Reviews** — the three testimonials are **examples**. Replace all three with **real** Google
      reviews (name + village), or remove the section. **Never publish invented reviews.**
- [ ] **Reply time** — the quote section says *"We aim to reply quickly."* Set a window you can
      actually stand behind (or leave it as-is; it's deliberately non-committal).
- [ ] **`priceRange` in schema** — the JSON-LD has `"priceRange": "££"` (a relative "moderate"
      tier, not a fixed price). Confirm that's fair, or delete the line.
- [ ] **Email** — currently `bookings@surreyhilltaxis.com` (the working address). Switch to
      `bookings@chiddingfoldtaxis.co.uk` once that mailbox exists (footer + JSON-LD `email`).

**Fill in the placeholders:**
- [ ] **Quote form** — create a free endpoint at [Formspree](https://formspree.io) (or Web3Forms/Basin)
      and replace `YOUR_FORM_ID` in the `<form action="…">`. Until then the submit button is disabled
      and the form tells people to call.
- [ ] **Google reviews link** — the reviews-section "Find us on Google" link points at the Google
      Maps listing; swap it for your live Google reviews URL once the profile is claimed.
- [ ] **Map coordinates** — the geo coordinates (`51.1858, -0.6015`) are approximate. Replace with
      the exact pin from the Google Business Profile (in the `<meta name="geo.position">`, the
      `ICBM` meta, and the JSON-LD `geo`).
- [ ] **Social profiles** — add real URLs to `sameAs: []` (JSON-LD) and the footer if you have them.
- [ ] **Photos** — add real/generated `hero.webp` + `about.webp` (the favicon/PWA icon set and the
      `og-image.jpg` social card are already generated and in the repo).

**Deliberately NOT claimed** (don't add these without proof): fixed prices/"cheapest", years
established, fleet size, DBS checks, insurance specifics, star ratings/review counts, or
guaranteed journey/pickup times. Add any of them only once they're verifiably true.
